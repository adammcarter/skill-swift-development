# Trusted Technical Resources

Use these sources when Swift behavior, Apple-platform guidance, or architectural tradeoffs are nuanced.

## Primary Canonical Sources

1. Apple Developer Documentation
   - Primary canonical source for Foundation, Swift, platform APIs, and official Apple guidance.
   - Home: https://developer.apple.com/

2. Apple WWDC Sessions And Sample Code
   - Primary canonical source for current recommended patterns, concurrency guidance, Swift evolution in practice, and intended Apple direction.
   - WWDC: https://developer.apple.com/videos/

3. Swift.org
   - Primary canonical source for language evolution, concurrency model details, and core Swift guidance.
   - Home: https://www.swift.org/

## Secondary Trusted Sources

Use these as secondary trusted sources when you need strong examples, practical implementation guidance, or experienced technical judgment beyond the canonical sources. They are valuable because of their consistently high code quality and technical depth, but they should not override Apple or Swift.org on factual behavior.

4. Hacking with Swift
   - Use as a secondary trusted source for practical Swift and SwiftUI implementation guidance when you need a clear example or fast problem-solving path.
   - Home: https://www.hackingwithswift.com/

5. Swift by Sundell
   - Use as a secondary trusted source for thoughtful Swift design, API-shape, architecture, and implementation tradeoff guidance.
   - Home: https://www.swiftbysundell.com/

6. Point-Free
   - Use as a secondary trusted source for thoughtful state, testing, and architecture guidance when deeper design judgment is needed.
   - Home: https://www.pointfree.co/

7. objc.io
   - Use as a secondary trusted source for high-quality Swift, Foundation, architecture, and engineering craft guidance.
   - Home: https://www.objc.io/

## How To Use Them

- Start with Apple documentation, WWDC, and Swift.org for platform truth, language semantics, concurrency behavior, and intended ecosystem direction.
- Use Hacking with Swift for practical examples, Swift by Sundell for design judgment, and Point-Free and objc.io for deeper architecture and testing guidance when canonical sources are not enough on their own.
- If sources disagree, prefer Apple, WWDC, and Swift.org for factual behavior and intended direction.
- Use secondary sources to improve clarity, examples, and implementation judgment rather than to override canonical Apple or Swift guidance.
